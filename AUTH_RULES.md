# 登录与权限规则梳理（Clerk / 境内登录 / 归档访问）

本文档用于固化当前讨论结论，作为后续改造与验收基线。

## 1. 背景与目标

- 目标一：登录与注册合并为单入口心智（用户只感知“登录/免费开始”）。
- 目标二：登录后可享 7 天归档能力。
- 目标三：未登录用户禁止查看归档（日历与归档详情均不可访问）。
- 目标四：前后端权限口径一致，避免“前端拦了但后端还能拿到数据”的漏洞。

## 2. 现状（As-Is）

- 系统存在 `guest / observer / tracker / admin` 四级 `tier`。
- 实际权限由 `effective tier` 决定：`observer/tracker` 到期后会降级为 `guest`。
- 新用户首次登录会分配 7 天 `observer`（试用）。
- 归档相关接口目前存在“guest 降级预览”逻辑。
- 归档日历接口当前未按登录态强限制，存在信息泄露窗口（可探测哪些日期有报告）。

## 3. 现有主要问题

- 问题1：登录/注册入口分散，用户决策成本高（`sign-in` 与 `sign-up` 双入口）。
- 问题2：归档模块缺少“后端强鉴权”闭环，存在旁路访问风险。
- 问题3：权限判断分散在多处（前端页面、后端路由、缓存），一致性差。
- 问题4：缓存键部分缺少权限维度，可能出现不同身份命中同一缓存数据。

## 4. 目标权限模型（To-Be）

采用“两层模型”统一表达：

1) 登录状态：`anonymous / signed_in`
2) 能力级别：`latest_preview / archive_7d / archive_all`

映射建议：

- 未登录：`latest_preview`
- 已登录且在 7 天内（试用/基础）：`archive_7d`
- 高阶订阅或管理员：`archive_all`
- 已登录但权益到期：回落到 `latest_preview`（但仍是已登录身份）

## 5. 页面权限矩阵（目标）

| 页面 | 未登录 | 已登录（7天） | 已登录（高阶） |
|---|---|---|---|
| `/reports/latest`（最新报告） | 允许预览 | 允许 | 允许 |
| `/reports`（归档页） | 禁止（跳登录） | 允许 | 允许 |
| `/reports/[date]`（归档详情） | 禁止（跳登录） | 允许（7天内范围） | 允许（全量范围） |
| `/dashboard`（个人中心） | 跳登录 | 允许 | 允许 |

## 6. API 权限矩阵（目标）

| API | 未登录 | 已登录（7天） | 已登录（高阶） |
|---|---|---|---|
| `GET /api/reports/latest/summary` | 允许预览 | 允许 | 允许 |
| `GET /api/reports/?scope=archive` | 401/403 | 允许（最近7天） | 允许（全量） |
| `GET /api/reports/calendar` | 401/403 | 允许（仅可见范围） | 允许（全量） |
| `GET /api/reports/date/{date}` | 401/403 | 允许（仅可见范围） | 允许（全量） |
| `GET /api/reports/{id}/full` | 401/403 | 允许（仅可见范围） | 允许（全量） |

说明：归档相关接口不再对未登录返回“降级预览”，统一改为拒绝访问。

## 7. 登录/注册入口策略（目标）

- 前端只保留一个主入口文案：`登录 / 免费开始`。
- 使用 Clerk 统一组件内完成“登录/注册切换”，不再在导航层提供两个分裂入口。
- 登录成功后统一回跳：优先 `redirect_url`，否则到默认页（建议 `/dashboard` 或触发来源页）。

## 8. 安全与一致性要求

- 所有归档权限以后端为准，前端只做体验层跳转与提示。
- 归档相关缓存键必须包含权限维度（至少包含 capability/tier）。
- 不允许通过直接拼 URL 获取未授权归档数据。
- 文案只对外表达“登录后 7 天可回溯”，避免暴露内部 `tier` 术语。

## 9. 实施清单（落地时按此执行）

- [ ] 前端路由守卫：未登录访问归档页面跳转登录并携带 `redirect_url`
- [ ] 导航入口收敛：登录/注册入口合并为单入口
- [ ] 后端归档接口鉴权：未登录统一 401/403
- [ ] 后端 7 天范围校验：对 `archive_7d` 生效
- [ ] 缓存键补齐权限维度，防止跨身份串缓存
- [ ] 联调验证（页面跳转 + API 直连 + URL 旁路）

## 10. 验收标准（最小可验）

- 未登录用户：
  - 访问 `/reports` 与 `/reports/[date]` 会被引导到登录；
  - 直接请求归档 API 返回 401/403；
  - 仍可访问 `/reports/latest` 预览。
- 新登录用户：
  - 登录后可访问归档；
  - 默认拥有最近 7 天归档范围。
- 7 天到期后：
  - 不再可访问归档接口与归档页面内容；
  - 仍可访问最新报告预览。
