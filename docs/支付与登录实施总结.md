# 支付接入 + 登录权限 — 实施总结

> 本文档为 PingPong 支付 + 账号权限的最终设计摘要，确认后按此实施。

---

## 一、决策汇总

| 项目 | 决策 |
|------|------|
| **支付渠道** | 先接 PingPong |
| **套餐** | 沿用 observer / tracker |
| **定价** | $29.9/月（统一） |
| **认证** | 继续用 Clerk，需检查并补齐 |
| **订阅记录** | 从简：用 User 表字段，不建新表 |

---

## 二、套餐与权限

| 套餐 | 月费 | 权限 |
|------|------|------|
| **guest** | 免费 | 预览 + 1 条报告列表 |
| **observer** | $29.9 | 完整报告内容 + 7 天历史 |
| **tracker** | $29.9 | 暂未开放 |

---

## 三、订阅记录 — 极简方案

不新建表，只使用现有 `User` 字段：

- `subscription_start`：订阅开始
- `subscription_end`：订阅结束（到期即视为 guest）
- `tier`：observer / tracker

支付成功后：
- 更新 `subscription_end = 当前时间 + 1 个月`
- 更新 `tier = observer`（tracker 暂未开放）

---

## 四、Clerk 合理性检查

### 4.1 现状

- **后端**：`middleware/auth.py` 已实现 Clerk JWT 验证、用户同步、tier 逻辑
- **前端**：未安装 `@clerk/nextjs`，AuthButtons 显示「认证已关闭」，Dashboard 传 `getToken = () => null`
- **环境**：`SKIP_CLERK=true`，相当于关闭校验

### 4.2 结论：继续用 Clerk 是合理的

| 维度 | 说明 |
|------|------|
| 成本 | 免费档足够当前用户量 |
| 能力 | 邮箱/Google 登录、JWT、Webhook 支持良好 |
| 工作量 | 后端已就绪，前端只需补 Clerk SDK 与路由保护 |
| 切换成本 | 换 Auth0/Supabase 需改 JWT 验证与用户映射逻辑 |

**需补充**：前端安装 `@clerk/nextjs`，配置 `ClerkProvider`、`getToken`、受保护路由。

## 五、技术架构

### 5.1 支付流程（PingPong）

```
用户选择套餐 → POST /api/payments/checkout
    → 后端创建 PingPong 收银台订单
    → 返回 checkout URL
    → 用户跳转 PingPong 支付页完成支付

PingPong 异步回调 → POST /api/payments/webhook
    → 验签
    → 更新 User.subscription_end、tier
    → 返回 200
```

### 5.2 权限流程

```
未登录 → 视为 guest → 仅预览 + 1 条
登录 + 未付费/已过期 → subscription_end 已过期 → 降为 guest
登录 + 已付费 observer → 7 天历史 + 完整内容
ADMIN_CLERK_IDS 或 tier=admin → 最高权限
```

## 七、实施阶段（建议顺序）

| 阶段 | 内容 | 产出 |
|------|------|------|
| **1** | 启用 Clerk + 前端登录 | 登录/登出、受保护路由、API 携带 Token |
| **2** | PingPong 沙箱对接 | 创建订单、返回 checkout URL |
| **3** | Webhook 处理 | 验签、更新 User.subscription_end、tier |
| **4** | 订阅 UI | 套餐选择（仅 observer）、支付跳转、成功/失败页 |

## 八、API 清单

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /api/payments/checkout` | POST | 创建 PingPong 订单，返回 checkout URL |
| `POST /api/payments/webhook` | POST | PingPong 异步通知（需验签） |
| `GET /api/users/me` | GET | 现有，返回 tier、subscription_end |
