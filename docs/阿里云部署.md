# 前后端一起部署到阿里云 ECS

在一台阿里云 ECS 上用 Docker Compose 同时跑前端和后端，开机自启。

---

## 一、购买与准备 ECS

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com)，创建实例：
   - **地域**：选和 RDS 同地域（访问数据库更快）。
   - **镜像**：**Ubuntu 22.04**。
   - **规格**：1 核 2G 起步即可，访问量大再升配。
   - **系统盘**：40GB 以上。
   - 设置 ** root 密码** 或 **密钥对**，记好以便 SSH。

2. **安全组**（必须放行，否则外网访问不了）：
   - 入方向：放行 **22**（SSH）、**80**（HTTP）、**443**（HTTPS）、**3000**（前端）、**8000**（后端）。
   - 若后面用 Nginx 反代，只通过 80/443 访问，可只放行 22、80、443，不放 3000/8000 更安全。

3. 用 SSH 连上 ECS（把 `你的公网IP` 换成 ECS 公网 IP）：
   ```bash
   ssh root@你的公网IP
   ```

---

## 二、安装 Docker 与 Docker Compose

在 ECS 上执行：

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker 并设为开机自启
systemctl start docker
systemctl enable docker

# 安装 Docker Compose 插件（与 Docker 一起用）
apt-get update && apt-get install -y docker-compose-plugin

# 验证
docker --version
docker compose version
```

---

## 三、上传代码并配置环境变量

任选一种方式把项目放到 ECS 上。

### 方式 A：Git 克隆（推荐）

```bash
# 若未装 git
apt-get install -y git

cd /opt   # 或你习惯的目录
git clone <你的仓库地址> moriqiquanHtml
cd moriqiquanHtml
```

### 方式 B：本地上传

在**你本机**打包后上传（在项目根目录执行）：

```bash
# 本机
tar --exclude=node_modules --exclude=.next --exclude=__pycache__ -czvf moriqiquanHtml.tar.gz .
scp moriqiquanHtml.tar.gz root@你的公网IP:/opt/
```

在 **ECS 上**解压：

```bash
mkdir -p /opt/moriqiquanHtml && cd /opt/moriqiquanHtml
tar -xzvf /opt/moriqiquanHtml.tar.gz -C /opt/moriqiquanHtml
```

---

### 配置 .env

在 ECS 上：

```bash
cd /opt/moriqiquanHtml
cp .env.example .env
nano .env   # 或 vim .env
```

**必填项**（把占位换成你的真实信息）：

```bash
# 数据库（阿里云 RDS 内网地址，与 ECS 同 VPC 时用内网更稳）
DATABASE_URL=postgresql://用户名:密码@RDS内网地址:5432/barkme

# 前端要请求的后端地址（浏览器用）
# 暂时用 IP 时：
NEXT_PUBLIC_API_URL=http://你的ECS公网IP:8000
# 后面用 Nginx + 域名时改成，例如：
# NEXT_PUBLIC_API_URL=https://你的域名/api

# 禁用 Clerk 时（不登录即最高权限）
NEXT_PUBLIC_SKIP_CLERK=true
SKIP_CLERK=true

# 若用 Clerk，取消注释并填写
# CLERK_PEM_PUBLIC_KEY=...
# CLERK_ISSUER=...
# NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=...
# CLERK_SECRET_KEY=...

# 后端 CORS（允许的前端来源，多个用逗号隔开）
# 用 IP 访问时：
CORS_ORIGINS=http://你的ECS公网IP:3000,http://你的ECS公网IP
# 用域名后改成：
# CORS_ORIGINS=https://你的域名
```

保存退出。

---

## 四、一键启动前后端

仍在 `/opt/moriqiquanHtml` 下执行：

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

首次会拉镜像、构建前端并启动，等 1～3 分钟。查看状态：

```bash
docker compose -f docker-compose.prod.yml ps
```

两个服务都是 **Up** 即正常。

- **前端**：浏览器访问 `http://你的ECS公网IP:3000`
- **后端 API**：`http://你的ECS公网IP:8000/docs` 可看接口文档

已配置 `restart: unless-stopped`，重启 ECS 后也会自动起来。

---

## 五、常用命令

```bash
cd /opt/moriqiquanHtml

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 只重启后端
docker compose -f docker-compose.prod.yml restart backend

# 只重启前端
docker compose -f docker-compose.prod.yml restart frontend

# 停止
docker compose -f docker-compose.prod.yml down

# 更新代码后重新构建并启动
git pull   # 若用 Git
docker compose -f docker-compose.prod.yml up -d --build
```

## 六、（可选）Nginx + 域名 + HTTPS

希望用域名访问、且 API 与前端同域时，可在同一台 ECS 装 Nginx，并申请免费 SSL（如阿里云 SSL 或 Let’s Encrypt）。

1. 安装 Nginx：
   ```bash
   apt-get install -y nginx
   ```

2. 新建站点配置（把 `你的域名` 换成实际域名，如 `report.example.com`）：
   ```bash
   nano /etc/nginx/sites-available/moriqiquan
   ```
   写入（前端 3000，后端 8000 反代到 `/api`，请求 `/api/xxx` 会转成后端 `/xxx`）：
   ```nginx
   server {
       listen 80;
       server_name 你的域名;
       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
       location /api/ {
           proxy_pass http://127.0.0.1:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```
   这样浏览器访问 `https://你的域名` 是前端，请求 `https://你的域名/api/...` 会转到后端。

3. 启用配置并重载 Nginx：
   ```bash
   ln -s /etc/nginx/sites-available/moriqiquan /etc/nginx/sites-enabled/
   nginx -t && systemctl reload nginx
   ```

4. 把域名解析到 ECS 公网 IP；若上 HTTPS，可用 `certbot` 或阿里云免费证书配置 `listen 443 ssl`。

5. 修改 `.env` 中的 `NEXT_PUBLIC_API_URL` 为 `https://你的域名`（不带 `/api`，前端会自动拼 `/api/reports` 等路径），然后**重新构建并启动前端**（该变量在构建时打进前端）：
   ```bash
   docker compose -f docker-compose.prod.yml up -d --build frontend
   ```
   之后访问 `https://你的域名` 即可，前端会请求同源的 `/api`，无需再暴露 3000/8000 端口。

## 七、小结

| 步骤       | 说明 |
|------------|------|
| 买 ECS     | Ubuntu 22.04，开放 22、3000、8000（或 80/443） |
| 装 Docker  | `curl -fsSL https://get.docker.com \| sh` + Docker Compose 插件 |
| 放代码     | Git clone 或本机打包 scp 上传 |
| 配 .env    | 至少填 `DATABASE_URL`、`NEXT_PUBLIC_API_URL`、`CORS_ORIGINS` |
| 启动       | `docker compose -f docker-compose.prod.yml up -d --build` |
| 访问       | 前端 `http://公网IP:3000`，后端 `http://公网IP:8000` |

这样前后端就一起在阿里云 ECS 上跑起来了，且会随机器重启自动运行。

## 八、更新代码未生效时的排查（如 mentat.hk 未显示最新代码）

按下面顺序逐项检查，确保「本地 → 远程仓库 → 服务器 → 容器」整条链路都是最新的。

### 1. 本地是否已提交并推送到远程

在**本机**项目目录执行：

```bash
git status          # 若有未提交的修改，先 commit
git log -1 --oneline   # 看当前最新 commit
git push            # 推送到远程（GitHub/Gitee 等）
```

- 若 `git status` 有大量 `M`（已修改未提交），需要先 `git add`、`git commit` 再 `git push`。
- 只有推送到远程后，服务器上的 `git pull` 才能拿到新代码。

### 2. 服务器上是否执行了 git pull

SSH 登录到 **mentat.hk 对应的 ECS**（或部署该项目的机器）：

```bash
ssh root@mentat.hk   # 或你的实际 SSH 地址
cd /opt/moriqiquanHtml   # 或你当时 clone 的目录
git status
git pull
```

- 若提示 `Already up to date` 但页面仍是旧的，继续看第 3、4 步。
- 若 `git pull` 报错（如冲突、权限、未配置 remote），先解决：`git remote -v` 确认仓库地址正确，必要时 `git fetch origin` 再 `git pull`。

### 3. 服务器上 pull 后是否重新构建并启动

拉完代码后必须**重新构建镜像并启动容器**，否则 Docker 仍在跑旧镜像：

```bash
cd /opt/moriqiquanHtml
docker compose -f docker-compose.prod.yml up -d --build
```

- `--build` 会按新代码重新 build 前端/后端镜像，再启动。
- 若只改了前端，可只重建前端：`docker compose -f docker-compose.prod.yml up -d --build frontend`。

### 4. 确认容器用的是新镜像

```bash
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml images
```

- 看容器创建时间是否为刚才 rebuild 之后；若时间很旧，说明可能没 rebuild 成功或用了别的 compose 文件。

### 5. 浏览器缓存与 CDN

- 浏览器强刷：`Ctrl+Shift+R`（Windows/Linux）或 `Cmd+Shift+R`（Mac）。
- 若前面有 CDN 或 Nginx 缓存，需在 Nginx 或 CDN 侧清理/禁用静态缓存，或等缓存过期。

### 6. 若服务器不是用 Git 部署的

若当时是用「本地上传 tar 包」方式部署的，服务器上没有 Git，则更新只能：

- 本机重新打包：`tar --exclude=node_modules --exclude=.next --exclude=__pycache__ -czvf moriqiquanHtml.tar.gz .`
- 上传到服务器：`scp moriqiquanHtml.tar.gz root@mentat.hk:/opt/`
- 在服务器上解压覆盖后重新构建：  
  `cd /opt/moriqiquanHtml && tar -xzvf /opt/moriqiquanHtml.tar.gz -C . --strip-components=0`（注意路径）  
  然后：`docker compose -f docker-compose.prod.yml up -d --build`

---

**快速自检表**

| 步骤 | 在哪里执行 | 命令/动作 |
|------|------------|-----------|
| ① 代码已推送 | 本机 | `git push`，在 GitHub/Gitee 上确认最新 commit |
| ② 服务器拉代码 | ECS (mentat.hk) | `cd /opt/moriqiquanHtml && git pull` |
| ③ 重新构建并启动 | ECS | `docker compose -f docker-compose.prod.yml up -d --build` |
| ④ 确认容器已更新 | ECS | `docker compose -f docker-compose.prod.yml ps` 看时间 |
| ⑤ 排除缓存 | 浏览器 | 强刷或无痕打开 `https://mentat.hk` |

## 九、阿里云与域名排查（IP 已是最新，域名仍显示老代码）

**你的情况**：`http://43.106.7.115/` 已是新代码，`mentat.hk` 打开仍是老代码。按下面顺序排查。

### 1. 确认域名解析到的 IP（必做）

在**本机**终端执行：

```bash
nslookup mentat.hk
# 或
dig mentat.hk +short
```

看返回的 **A 记录 IP** 是否等于 **43.106.7.115**。

| 结果 | 说明 | 处理 |
|------|------|------|
| 不是 43.106.7.115 | 域名指到了别的机器（旧服务器/错误 IP） | 去域名服务商改 A 记录，见下文「2. 修改域名解析」 |
| 是 43.106.7.115 | 解析正确，问题在缓存或同一台机上的不同配置 | 做「3. 解析正确时的排查」 |

### 2. 修改域名解析（解析不对时）

- **若域名在阿里云**：  
  [阿里云控制台](https://dc.console.aliyun.com) → **域名** → 找到 **mentat.hk** → **解析设置** → 把 **A 记录** 的「记录值」改为 **43.106.7.115**，保存。  
  等 TTL 生效（通常几分钟到几小时，可看解析页上的 TTL 值）。

- **若域名不在阿里云**（如在 GoDaddy、Cloudflare、Namecheap 等）：  
  到该服务商的 DNS 管理里，把 **mentat.hk**（或 `@`）的 **A 记录** 改为 **43.106.7.115**。

改完后可再次 `nslookup mentat.hk` 或 `dig mentat.hk +short` 确认已变为 43.106.7.115。

### 3. 解析正确（已是 43.106.7.115）时的排查

说明访问的都是同一台 ECS，差异来自「通过域名」的路径上有缓存或不同配置。

**① 是否接了 CDN**

- **阿里云 CDN**：  
  [CDN 控制台](https://cdn.console.aliyun.com) → 找到对应域名 → **刷新预热** → 提交 **URL 刷新**（如 `https://mentat.hk/`）或 **目录刷新**（如 `https://mentat.hk/`），清除缓存后再用浏览器无痕访问 mentat.hk 试一次。
- **Cloudflare / 其他 CDN**：  
  在控制台里做 **Purge Cache** 或类似「清除缓存」，再测。

**② 同一台 ECS 上 Nginx 是否按域名区分**

若你用 Nginx 反代，且有多条 `server`（例如一条用 IP、一条用 `server_name mentat.hk`），要确认：

- mentat.hk 的 `server` 里 `proxy_pass` 指向的也是本机 3000/8000，和用 IP 访问时一致。
- 没有给 mentat.hk 单独配了很长的静态缓存（如 `proxy_cache_valid 200 7d`），否则会长期是旧页面。

SSH 上 ECS 后：

```bash
grep -r "mentat.hk\|43.106.7.115" /etc/nginx/
```

看两个是否都反代到同一组 upstream/端口。

**③ 浏览器与本地缓存**

- 用**无痕/隐私模式**或换一台设备访问 `https://mentat.hk`。
- 或强刷：`Ctrl+Shift+R` / `Cmd+Shift+R`。

### 4. 阿里云侧自查清单

| 检查项 | 在哪里看 | 要确认的 |
|--------|----------|-----------|
| 域名解析 | 阿里云 → 域名 → 解析设置 | mentat.hk 的 A 记录 = 43.106.7.115 |
| 安全组 | ECS → 实例 → 安全组 | 80/443 已放行（若用 Nginx）；22 已放行 |
| CDN（若启用） | CDN 控制台 → 刷新预热 | 对 mentat.hk 做过 URL/目录刷新 |
| 同一台 ECS | 用 IP 访问 43.106.7.115 | 已确认是新代码，说明应用本身已更新 |

### 5. 小结：你当前最该做的两步

1. **本机执行** `nslookup mentat.hk`（或 `dig mentat.hk +short`），看结果是否是 **43.106.7.115**。  
   - 不是 → 去域名服务商把 mentat.hk 的 A 记录改成 43.106.7.115。  
   - 是 → 做第 2 步。
2. **若解析已是 43.106.7.115**：去阿里云 CDN（若用了）做一次刷新；并用浏览器无痕访问 mentat.hk 再试。
