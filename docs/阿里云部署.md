# 前后端一起部署到阿里云 ECS

在一台阿里云 ECS 上用 Docker Compose 同时跑前端和后端，开机自启。

---

## 一、购买与准备 ECS

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com)，创建实例：
   - **地域**：选和 RDS 同地域（访问数据库更快）。
   - **镜像**：**Ubuntu 22.04**。
   - **规格**：1 核 2G 起步即可，访问量大再升配。
   - **系统盘**：40GB 以上。
   - 设置 ** root 密码** 或 **密钥对**，记好以便 SSH。

2. **安全组**（必须放行，否则外网访问不了）：
   - 入方向：放行 **22**（SSH）、**80**（HTTP）、**443**（HTTPS）、**3000**（前端）、**8000**（后端）。
   - 若后面用 Nginx 反代，只通过 80/443 访问，可只放行 22、80、443，不放 3000/8000 更安全。

3. 用 SSH 连上 ECS（把 `你的公网IP` 换成 ECS 公网 IP）：
   ```bash
   ssh root@你的公网IP
   ```

---

## 二、安装 Docker 与 Docker Compose

在 ECS 上执行：

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker 并设为开机自启
systemctl start docker
systemctl enable docker

# 安装 Docker Compose 插件（与 Docker 一起用）
apt-get update && apt-get install -y docker-compose-plugin

# 验证
docker --version
docker compose version
```

---

## 三、上传代码并配置环境变量

任选一种方式把项目放到 ECS 上。

### 方式 A：Git 克隆（推荐）

```bash
# 若未装 git
apt-get install -y git

cd /opt   # 或你习惯的目录
git clone <你的仓库地址> moriqiquanHtml
cd moriqiquanHtml
```

### 方式 B：本地上传

在**你本机**打包后上传（在项目根目录执行）：

```bash
# 本机
tar --exclude=node_modules --exclude=.next --exclude=__pycache__ -czvf moriqiquanHtml.tar.gz .
scp moriqiquanHtml.tar.gz root@你的公网IP:/opt/
```

在 **ECS 上**解压：

```bash
mkdir -p /opt/moriqiquanHtml && cd /opt/moriqiquanHtml
tar -xzvf /opt/moriqiquanHtml.tar.gz -C /opt/moriqiquanHtml
```

---

### 配置 .env

在 ECS 上：

```bash
cd /opt/moriqiquanHtml
cp .env.example .env
nano .env   # 或 vim .env
```

**必填项**（把占位换成你的真实信息）：

```bash
# 数据库（阿里云 RDS 内网地址，与 ECS 同 VPC 时用内网更稳）
DATABASE_URL=postgresql://用户名:密码@RDS内网地址:5432/barkme

# 前端要请求的后端地址（浏览器用）
# 暂时用 IP 时：
NEXT_PUBLIC_API_URL=http://你的ECS公网IP:8000
# 后面用 Nginx + 域名时改成，例如：
# NEXT_PUBLIC_API_URL=https://你的域名/api

# 禁用 Clerk 时（不登录即最高权限）
NEXT_PUBLIC_SKIP_CLERK=true
SKIP_CLERK=true

# 若用 Clerk，取消注释并填写
# CLERK_PEM_PUBLIC_KEY=...
# CLERK_ISSUER=...
# NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=...
# CLERK_SECRET_KEY=...

# 后端 CORS（允许的前端来源，多个用逗号隔开）
# 用 IP 访问时：
CORS_ORIGINS=http://你的ECS公网IP:3000,http://你的ECS公网IP
# 用域名后改成：
# CORS_ORIGINS=https://你的域名
```

保存退出。

---

## 四、一键启动前后端

仍在 `/opt/moriqiquanHtml` 下执行：

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

首次会拉镜像、构建前端并启动，等 1～3 分钟。查看状态：

```bash
docker compose -f docker-compose.prod.yml ps
```

两个服务都是 **Up** 即正常。

- **前端**：浏览器访问 `http://你的ECS公网IP:3000`
- **后端 API**：`http://你的ECS公网IP:8000/docs` 可看接口文档

已配置 `restart: unless-stopped`，重启 ECS 后也会自动起来。

---

## 五、常用命令

```bash
cd /opt/moriqiquanHtml

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 只重启后端
docker compose -f docker-compose.prod.yml restart backend

# 只重启前端
docker compose -f docker-compose.prod.yml restart frontend

# 停止
docker compose -f docker-compose.prod.yml down

# 更新代码后重新构建并启动
git pull   # 若用 Git
docker compose -f docker-compose.prod.yml up -d --build
```

## 六、（可选）Nginx + 域名 + HTTPS

希望用域名访问、且 API 与前端同域时，可在同一台 ECS 装 Nginx，并申请免费 SSL（如阿里云 SSL 或 Let’s Encrypt）。

1. 安装 Nginx：
   ```bash
   apt-get install -y nginx
   ```

2. 新建站点配置（把 `你的域名` 换成实际域名，如 `report.example.com`）：
   ```bash
   nano /etc/nginx/sites-available/moriqiquan
   ```
   写入（前端 3000，后端 8000 反代到 `/api`，请求 `/api/xxx` 会转成后端 `/xxx`）：
   ```nginx
   server {
       listen 80;
       server_name 你的域名;
       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
       location /api/ {
           proxy_pass http://127.0.0.1:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```
   这样浏览器访问 `https://你的域名` 是前端，请求 `https://你的域名/api/...` 会转到后端。

3. 启用配置并重载 Nginx：
   ```bash
   ln -s /etc/nginx/sites-available/moriqiquan /etc/nginx/sites-enabled/
   nginx -t && systemctl reload nginx
   ```

4. 把域名解析到 ECS 公网 IP；若上 HTTPS，可用 `certbot` 或阿里云免费证书配置 `listen 443 ssl`。

5. 修改 `.env` 中的 `NEXT_PUBLIC_API_URL` 为 `https://你的域名`（不带 `/api`，前端会自动拼 `/api/reports` 等路径），然后**重新构建并启动前端**（该变量在构建时打进前端）：
   ```bash
   docker compose -f docker-compose.prod.yml up -d --build frontend
   ```
   之后访问 `https://你的域名` 即可，前端会请求同源的 `/api`，无需再暴露 3000/8000 端口。

## 七、小结

| 步骤       | 说明 |
|------------|------|
| 买 ECS     | Ubuntu 22.04，开放 22、3000、8000（或 80/443） |
| 装 Docker  | `curl -fsSL https://get.docker.com \| sh` + Docker Compose 插件 |
| 放代码     | Git clone 或本机打包 scp 上传 |
| 配 .env    | 至少填 `DATABASE_URL`、`NEXT_PUBLIC_API_URL`、`CORS_ORIGINS` |
| 启动       | `docker compose -f docker-compose.prod.yml up -d --build` |
| 访问       | 前端 `http://公网IP:3000`，后端 `http://公网IP:8000` |

这样前后端就一起在阿里云 ECS 上跑起来了，且会随机器重启自动运行。
