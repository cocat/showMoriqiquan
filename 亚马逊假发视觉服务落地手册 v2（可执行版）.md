# 亚马逊假发视觉服务落地手册 v2（可执行版）

> 版本说明：本版按 2026-04-03 可核实信息重构，目标不是“写得热闹”，而是“更容易成交、交付、复购、扩张”。

---

# 第一部分：项目定位

## 1.1 一句话定位

**我们卖的不是“纯 AI 出图”，而是“基于真实产品素材的亚马逊视觉提效服务”。**

更具体一点：

```
你提供真实产品素材
→ 我们帮你做统一风格、统一模特、统一场景的电商视觉
→ 帮你更快上新、更稳测试、更低成本迭代
```

## 1.2 服务边界

### 可以主推的交付内容（低风险）

| 交付类型 | 能不能做 | 风险 | 备注 |
|:---|:---:|:---:|:---|
| 副图场景图 | ✅ | 低 | 最适合切入 |
| 信息图 | ✅ | 低 | 参数、长度、密度、帽型 |
| 视频封面 | ✅ | 低 | 适合配合广告素材 |
| 多场景展示图 | ✅ | 低 | 用于详情页、Brand Store |
| 多肤色展示图 | ✅ | 中 | 必须注明为展示用途 |
| A+ 横幅草图 | ✅ | 中 | 正式上线需客户具备品牌资格 |
| Brand Store 视觉图 | ✅ | 中 | 适合品牌客户 |

### 条件可做的交付内容（中风险）

| 交付类型 | 能不能做 | 风险 | 条件 |
|:---|:---:|:---:|:---|
| 主图增强修图 | ✅ | 中 | 必须基于真实产品图或真实佩戴图 |
| 白底佩戴图 | ⚠️ | 中高 | 先做测试，不默认承诺可稳定上架 |
| 多角度模特图 | ✅ | 中 | 需保证原始素材完整 |

### 不建议作为核心卖点的内容（高风险）

| 交付类型 | 建议 | 原因 |
|:---|:---:|:---|
| 纯 AI 生成主图 | 不主推 | 与 Amazon “actual product”要求存在冲突 |
| 纯 AI 发际线 close-up | 不主推 | 极易与实物不符 |
| 纯 AI 色卡图 | 不主推 | 色差风险大，容易引发退货 |
| 纯 AI “无痕蕾丝”承诺图 | 不主推 | 容易过度承诺 |

## 1.3 最适合切入的价值主张

```
客户最容易买单的不是“AI 很厉害”
而是下面四句：

  • 上新更快
  • 风格更统一
  • 素材复用率更高
  • 比真人拍摄便宜很多
```

# 第二部分：目标客户与筛选

## 2.1 理想客户画像（ICP）

优先做这三类客户：

### A 类：月上新频率高的中小卖家

```
特征：
  • 每月持续上新
  • 预算有限
  • 没有成熟拍摄团队
  • 愿意先试 1-3 款

适合卖：
  • 单 SKU 试单
  • 批量上新包
```

### B 类：有品牌意识的成熟卖家

```
特征：
  • 已有稳定 SKU
  • 图片风格不统一
  • 有 A+ / Brand Store / 广告投放需求

适合卖：
  • 品牌视觉统一包
  • 月度托管
```

### C 类：工厂型跨境客户

```
特征：
  • 产品强
  • 视觉弱
  • 希望从“供货思维”升级到“品牌思维”

适合卖：
  • 标准化上新包
  • 品牌升级包
```

## 2.2 不优先碰的客户

```
  • 完全没有标准化素材的
  • 只想 50-80 元拿高质量全套图的
  • 希望你承诺“百分百能过 Amazon 主图审核”的
  • SKU 很少、又没有长期计划的
  • 不愿意配合提供真实细节图的
```

## 2.3 线索打分表

每个客户按 1-5 分打分，总分越高越先联系：

| 维度 | 评分说明 |
|:---|:---|
| Brand Registry | 有为 5 分，无为 1 分 |
| SKU 数量 | >50 为 5 分，<10 为 1 分 |
| 上新频率 | 每周上新更高分 |
| 当前图片质量 | 一般但可提升的更高分 |
| 是否找到决策人 | 老板/运营负责人更高分 |
| 是否愿意试单 | 愿意先做 1-3 款更高分 |

# 第三部分：产品设计与报价建议

## 3.1 产品结构

### 产品 1：审图诊断

```
适合对象：
  • 对你不熟的冷线索

交付：
  • 1 份图片问题清单
  • 1 页视觉改造建议

作用：
  • 建立专业度
  • 为试单成交做铺垫
```

### 产品 2：单 SKU 付费试单

```
适合对象：
  • 首次合作客户

建议交付：
  • 4-6 张图
  • 含 1 张信息图
  • 含 1 个视频封面或短转场视频

作用：
  • 先成交
  • 先验证素材质量和沟通效率
```

### 产品 3：批量上新包

```
适合对象：
  • 上新频率高的店铺

建议交付：
  • 5-20 个 SKU
  • 统一模特 / 统一场景 / 统一模板

作用：
  • 提利润
  • 做标准化
```

### 产品 4：品牌视觉统一包

```
适合对象：
  • 品牌卖家

建议交付：
  • 品牌模特设定
  • 视觉风格板
  • Listing 副图模板
  • A+ 横幅草图
  • Brand Store 视觉图

作用：
  • 提高客单价
  • 提升复购和长期合作概率
```

### 产品 5：月度托管

```
适合对象：
  • 已验证合作顺畅的老客户

建议交付：
  • 每月固定 SKU 数
  • 固定交付时效
  • 固定场景模板

作用：
  • 稳定现金流
  • 提高团队效率
```

## 3.2 报价建议

> 以下为建议区间，不是唯一答案。你的报价应根据素材质量、修改次数、时效要求调整。

| 产品 | 建议价 |
|:---|:---|
| 审图诊断 | 99-299 元 / 次 |
| 单 SKU 试单 | 299-699 元 / 款 |
| 批量上新包 | 399-999 元 / 款 |
| 品牌视觉统一包 | 3000-10000 元 / 项目 |
| 月度托管 | 5000-30000 元 / 月 |

## 3.3 报价必须说明的条款

```
必须写清楚：
  • 默认包含几张图
  • 默认包含几次修改
  • 交付时间是多少
  • 加急费怎么算
  • 多肤色 / 多场景 / 多比例是否额外收费
  • 客户是否授权你公开案例
  • 纯展示图与可上架图的区别
```

# 第四部分：客户素材输入 SOP

## 4.1 每个 SKU 必须提交的素材

```
  □ 正面图
  □ 左侧 45° 图
  □ 右侧 45° 图
  □ 背面图
  □ 头顶分缝图
  □ 发际线 close-up
  □ lace close-up
  □ 长度尺图
  □ 自然光色泽图
  □ 如需展示包装，补包装图
```

## 4.2 拍摄要求

```
  • 尽量白墙或纯净背景
  • 光线均匀，不要强黄光
  • 不要开美颜
  • 不要压缩原图
  • 同一款保持固定距离
  • 发尾、头顶、侧边都要清楚
```

## 4.3 客户不达标素材的处理规则

```
如果素材不达标：
  • 先返回素材整改意见
  • 或者只提供“展示参考图”
  • 不承诺直接上架效果
```

# 第五部分：生产流程 SOP

## 5.1 标准流程

```
Step 1：接单与资格预审
Step 2：收素材并检查完整度
Step 3：给出视觉方向和场景建议
Step 4：先做 1 张样图确认风格
Step 5：批量出图
Step 6：内部 QA
Step 7：客户确认与修改
Step 8：打包交付
Step 9：跟踪上线和效果
```

## 5.2 每单必须产出的内部文件

```
  • 客户信息卡
  • SKU 素材清单
  • 风格说明
  • 交付清单
  • 修改记录
  • 案例授权状态
```

## 5.3 推荐的交付目录结构

```text
客户名/
  SKU001/
    source/
    draft/
    final_listing/
    final_showcase/
  SKU002/
  交付说明.md
```

# 第六部分：QA 验收标准

## 6.1 每张图交付前必须检查

```
  □ 发量一致
  □ 长度一致
  □ 卷度一致
  □ 分缝位置一致
  □ 发际线不过假
  □ lace 与皮肤融合自然
  □ 色泽不偏离原图
  □ 模特五官、耳朵、手指正常
  □ 光照方向统一
  □ 服饰不抢主体
  □ 文案参数无误
  □ 文件尺寸符合渠道要求
```

## 6.2 分级处理规则

```
A 级：可直接交付
B 级：可展示，但不建议上架
C 级：需返工
```

# 第七部分：获客与成交流程

## 7.1 获客渠道优先级

### 第一优先：亚马逊反查 + 精准触达

```
做法：
  • 先建 50 家目标客户池
  • 再按打分筛成前 10 家重点对象
  • 只给高分客户做定制触达
```

### 第二优先：1688 / 产业带

```
做法：
  • 重点找许昌、青岛
  • 优先联系明显在做 Amazon 供货的工厂
```

### 第三优先：微信群 / 论坛 / 线下活动

```
原则：
  • 不先打广告
  • 先建立专业感
  • 再用案例私聊推进
```

## 7.2 第一轮触达脚本

### 中小卖家版

```
你好，看到你们在做 Amazon 假发。

我们这边做的是“基于真实素材的电商视觉提效”，
能帮你把同一个 SKU 做成统一风格的副图、场景图、信息图，
比真人拍摄便宜很多，也更适合批量上新。

如果你愿意，我可以先免费帮你看 1 个链接，
直接告诉你现在图片最影响点击和转化的点在哪。
```

### 品牌卖家版

```
你好，我看了你们店里几个核心 SKU，
产品本身不错，但图片风格还不够统一。

我们现在专门帮假发卖家做“统一视觉 + 快速上新”方案，
尤其适合有多个长度、多个颜色、多个系列的品牌店。

如果方便，我可以先给你做一页视觉诊断，
把最值得改的 3 个位置标出来。
```

## 7.3 成交流程

```
冷触达
→ 免费审图
→ 单 SKU 试单
→ 3-5 SKU 小批量
→ 月度托管
```

# 第八部分：小红书内容策略

## 8.1 账号配置

**起步阶段先做 2 个号，不建议一开始就铺矩阵。**

```
账号 1：品牌号
  • 内容：案例、流程、交付前后对比

账号 2：专家号
  • 内容：审图思路、素材拍摄指南、Amazon 图片合规边界
```

## 8.2 内容优先级

### 最值得发的内容

```
  • 同一 SKU 从原始素材到完整副图体系
  • 一个客户如何把多个 SKU 做成统一视觉
  • 什么样的素材最容易出可用图
  • Amazon 图片最常见的 5 个坑
  • 为什么有些图适合详情页，不适合主图
```

### 少发或慎发的内容

```
  • “逆天”“炸裂”“骗过真人”式标题
  • 让人误会是真人实拍的表达
  • 未经授权展示客户图
  • 未标识 AI 生成属性的强对比案例
```

## 8.3 小红书内容节奏

```
每周 4 篇即可：
  • 2 篇案例
  • 1 篇干货
  • 1 篇流程 / 客户 FAQ
```

# 第九部分：4 周落地计划

## Week 1：建立可信样本

```
  □ 自购 2-3 顶样品或拿到可授权样品
  □ 建好 6 组标准案例
  □ 做 2 套不同风格模板
  □ 做 1 份审图诊断模板
  □ 建客户线索表和打分表
```

## Week 2：建立线索池并试触达

```
  □ 整理 50 家目标客户
  □ 筛出前 10 家高分客户
  □ 发第一轮精准触达
  □ 小红书发布 4 篇内容
  □ 加 2-3 个行业群
```

## Week 3：拿下首批试单

```
  □ 推进 5-10 个审图对话
  □ 成交 2-3 个付费试单
  □ 建标准交付表和修改记录表
  □ 收集客户真实反馈
```

## Week 4：形成复购模型

```
  □ 把试单客户推进到小批量
  □ 复盘哪个场景最容易成交
  □ 复盘哪类素材最容易出问题
  □ 产出 2 个可公开案例
  □ 开始谈月度托管
```

# 第十一部分：风险与合规

## 11.1 Amazon 侧

```
  • 主图必须优先满足 actual product 原则
  • 不要承诺“纯 AI 主图稳定通过”
  • A+ 交付前先确认 Brand Registry
  • 尽量把 AI 强项放在副图、广告、Store、视频封面
```

## 11.2 内容发布侧

```
  • 2025-09-01 起，中国《人工智能生成合成内容标识办法》已实施
  • 展示 AI 图时，要考虑显式标识和用户声明
  • 不要用“真人实拍”误导纯 AI 图
```

## 11.3 商务侧

```
  • 案例公开前要确认授权
  • 竞争品牌图不要直接拿来当商用案例
  • 交付范围、修改次数、可公开范围都要提前写清楚
```

# 第十二部分：一句话执行原则

```
先卖“专业诊断”和“标准化流程”
再卖“出图速度”
最后才卖“AI 很厉害”
```

## 12.1 这版手册最重要的结论

```
  • 先把服务边界说清楚
  • 先做副图和统一视觉，不要先赌主图
  • 先做付费试单，不要大量免费试
  • 先接高分客户，不要平均用力
  • 先拿数据，再谈更高价格
```
